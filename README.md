
# Практическое задание  

### Тема
Классы, указатели, динамическая память, работа с объектами через указатели.

### Цель работы
Научиться:
- создавать собственные классы;
- работать с указателями на объекты;
- использовать динамическое выделение памяти (`new`, `delete`);
- вызывать методы объектов через указатель (`->`);
- организовывать простое консольное меню.

---

## Условие задания

Недавно вам проводилась экскурсия по офису Яндекса. Во время экскурсии гид(это я :) ) показывал разные помещения: переговорные, кухни, зоны отдыха, рабочие пространства и другие комнаты.

Необходимо написать программу на C++, которая моделирует такую экскурсию.

В программе должны использоваться два класса:

- `Room` — комната в офисе;
- `Guide` — гид, который проводит экскурсию.

Также необходимо реализовать консольный интерфейс, позволяющий:
1. посещать комнаты;
2. узнавать информацию о текущей комнате;
3. выводить список всех посещённых комнат;
4. определять самую посещаемую комнату;
5. завершать работу программы.

---

## Требования к программе

### 1. Класс `Room`

Класс `Room` должен описывать одну комнату офиса.

#### Поля класса:
- `name` — название комнаты;
- `type` — тип комнаты;
- `capacity` — вместимость комнаты;
- `visitCount` — количество посещений комнаты.

#### Методы класса:
- конструктор для инициализации всех основных полей;
- `printInfo()` — выводит информацию о комнате;
- `visit()` — увеличивает количество посещений на 1;
- `getVisitCount()` — возвращает количество посещений;
- `getName()` — возвращает название комнаты.

---

### 2. Класс `Guide`

Класс `Guide` должен описывать гида, который проводит студентов по офису.

#### Поля класса:
- `name` — имя гида;
- `currentRoom` — указатель на текущую комнату (`Room*`).

#### Методы класса:
- конструктор;
- `goToRoom(Room* room)` — переводит группу в указанную комнату:
  - делает эту комнату текущей;
  - увеличивает счётчик её посещений;
- `printCurrentRoomInfo()` — выводит информацию о текущей комнате;
- `getCurrentRoom()` — возвращает указатель на текущую комнату.

---

## Требования к основной программе

В функции `main()` необходимо:

1. Создать **5 комнат динамически** с помощью `new`.
2. Сохранить указатели на созданные комнаты в массиве указателей.
3. Создать объект класса `Guide`.
4. Реализовать консольное меню, которое работает в цикле до тех пор, пока пользователь не выберет пункт «Выход».
5. Перед завершением программы освободить всю выделенную память с помощью `delete`.

---

## Обязательный функционал меню

Программа должна поддерживать следующие действия.

### 1. Посетить комнату
Пользователь выбирает одну из доступных комнат по номеру.  
После этого гид переводит группу в выбранную комнату.

При посещении комнаты:
- поле `currentRoom` должно обновляться;
- у комнаты должен увеличиваться счётчик посещений.

---

### 2. Показать информацию о текущей комнате
Программа должна вывести информацию о комнате, в которой группа находится в данный момент.

Если экскурсия ещё не началась и текущая комната не выбрана, нужно вывести сообщение:

`Группа пока не находится ни в одной комнате.`

---

### 3. Показать все посещённые комнаты
Программа должна вывести список всех комнат, у которых количество посещений больше 0.

Для каждой комнаты рекомендуется выводить:
- название;
- тип;
- вместимость;
- количество посещений.

Если ни одна комната ещё не посещалась, нужно вывести сообщение:

`Пока не посещено ни одной комнаты.`

---

### 4. Найти самую посещаемую комнату
Программа должна определить комнату с максимальным значением `visitCount` и вывести:
- её название;
- количество посещений.

Если ни одна комната ещё не посещалась, нужно вывести сообщение:

`Пока нет посещённых комнат.`

Если несколько комнат имеют одинаковое максимальное число посещений, можно вывести первую найденную.

---

### 5. Выход
Программа завершает работу.  
Перед завершением необходимо освободить память, выделенную под комнаты.

---

## Ограничения и рекомендации

### Обязательно:
- использовать классы `Room` и `Guide`;
- использовать указатель `Room* currentRoom` в классе `Guide`;
- создавать объекты комнат через `new`;
- обращаться к объектам через указатель с помощью оператора `->`;
- освобождать память через `delete`.

### Разрешается:
- хранить комнаты в обычном массиве указателей, например:
  ```cpp
  Room* rooms[5];
  ```
- не использовать контейнеры STL (`vector`, `map` и т. д.);
- не реализовывать сложную проверку некорректного ввода.

## Подсказки

1. Для хранения текущей комнаты в классе `Guide` используйте указатель:
   ```cpp
   Room* currentRoom;
   ```

2. Если текущая комната ещё не выбрана, указатель можно установить в `nullptr`.

3. При вызове метода у объекта через указатель используйте оператор:
   ```cpp
   room->printInfo();
   ```

4. Для поиска самой посещаемой комнаты удобно пройти по массиву комнат и найти объект с максимальным `visitCount`.

5. Не забудьте удалить все созданные объекты:
   ```cpp
   delete rooms[i];
   ```
